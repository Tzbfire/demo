# 讲者备注｜什么是 Coding Agent？尤其什么是 Codex

总时长建议：30 分钟；节奏：15 页 × 平均 2 分钟。

## 01. 封面｜什么是 Coding Agent？ / 尤其什么是 Codex

**建议用时**：1 分钟

开场先降低技术门槛：今天不是教大家写代码，而是解释一种新同事——它能把“我要一个结果”拆成步骤，并动手在安全环境里完成。强调：听完要知道能用它做什么、怎么提需求、哪里必须人工把关。

## 02. 为什么现在要懂｜它不是程序员专属工具，而是「重复工作自动化」的新入口

**建议用时**：2 分钟

用白领熟悉的场景切入：周报、Excel、合同整理、客户名单清洗。过去 AI 像顾问；Coding Agent 更像能接任务的实习生。关键变化不是“人人当程序员”，而是很多办公流程可以被做成可重复、可检查的小工具。

## 03. 一句话定义｜Agent = 会围绕目标反复行动的 AI

**建议用时**：2 分钟

不要陷入术语。Agent 的本质是“闭环”：它不是只说一句答案，而是会看反馈继续做。Coding Agent 的工作对象是代码和数字文件，但业务人员要理解的是：它可以把模糊目标变成一套可运行流程。

## 04. Agent 闭环｜一个 Coding Agent 的基本闭环

**建议用时**：2 分钟

这页讲“为什么它像同事而不像搜索框”。每一步都可以对应办公室工作：理解需求、找资料、写初稿、检查格式、提交给主管。代码只是它的执行媒介；真正重要的是人类仍然定义目标和验收标准。

## 05. 它具体做什么｜Coding Agent 具体在做什么？

**建议用时**：2 分钟

这页把“写代码”拆开。很多非技术同事以为写代码就是敲字，其实软件工作大量时间在读上下文、测试、修错、解释。Coding Agent 的价值是能把这些琐碎循环跑起来。

## 06. Codex 是什么｜Codex 是 OpenAI 的 coding agent

**建议用时**：2 分钟

这里要澄清：Codex 这个名字以前也指过代码模型，现在重点是 OpenAI 的编码代理产品/体系。对非技术听众可以说：它是 ChatGPT 家族里更会“动手做软件任务”的那一类能力。

## 07. Codex vs ChatGPT｜ChatGPT 更像顾问，Codex 更像执行者

**建议用时**：2 分钟

不要把两者对立起来。ChatGPT 帮你想清楚“要什么”，Codex 更适合接“把这个做出来”的任务。非技术人员可以先用 ChatGPT 梳理需求，再把清楚的任务交给 Coding Agent 或技术同事。

## 08. 白领场景｜非技术白领可以从哪些场景开始？

**建议用时**：2 分钟

强调“低风险、重复、高频、规则清楚”的任务最适合。不要一上来要求它改核心系统。一个好开端是：本来每周都要人工复制粘贴的流程。

## 09. 会写任务比会写代码重要｜你不一定要会写代码，但要会写「可验收的任务」

**建议用时**：2 分钟

这是给白领最关键的能力。很多失败不是 AI 不行，而是任务没有边界：输入在哪、输出长什么样、什么算完成都没说。用项目经理的语言提需求，比用程序员语言更重要。

## 10. Prompt 模板｜给 Coding Agent 的任务模板

**建议用时**：2 分钟

可以现场让大家拍照。讲重点：材料和验收最重要。不要只说“帮我做个自动化”，而要说“输入是什么、输出是什么、不能碰什么、怎么判断对”。

## 11. 一个办公室例子｜例子：把「每周销售表」变成自动报告

**建议用时**：2 分钟

用这个例子把抽象能力落地。它不是替代财务/销售判断，而是把机械部分自动化：清洗、汇总、出图、生成初稿。人负责业务口径和最后解释。

## 12. 人机分工｜人类不是被替代，而是从「执行者」变成「验收者 / 设计者」

**建议用时**：2 分钟

这页很适合对管理层讲：组织要重新定义分工。业务同事不是直接上生产系统，而是学会把需求变成可验收任务；技术同事从“所有小需求都亲手做”转向审核、平台化和治理。

## 13. 安全边界｜安全不是最后补丁，而是使用方式的一部分

**建议用时**：2 分钟

引用官方安全思路：Codex 有沙盒、审批、网络控制等机制，但企业使用仍要建立自己的边界。非技术同事的简单规则：真实敏感数据不要直接丢给工具；让它先在测试材料上做；上线前必须有人审。

## 14. 一周试点｜企业内试点：从一个低风险重复任务开始

**建议用时**：2 分钟

给可执行的落地建议：不要先开大会谈战略，先找一个低风险任务试点。成功的标志不是炫酷，而是下周能不能复用、别人能不能照模板提需求。

## 15. 总结｜最后带走三句话

**建议用时**：1 分钟

收束：这不是一门编程课，而是一次工作方式更新。鼓励听众回去想一个本周就能试的小任务：重复、低风险、输入输出明确。
